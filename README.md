# 100Hires AI Tools Project

## What This Is
A hands-on exploration of AI assisted development tools as part of the 100Hires application process.  
Not a finished product, more of a documented learning sprint.

---

## Setup
**Tools used:**
- Cursor IDE
- Claude Code Extension
- Codex Extension

---

## What I Actually Did
1. Set up Cursor IDE and configured Claude Code + Codex extensions
2. Created and connected a GitHub repository
3. Opened the repo inside Cursor and poked around
4. Tested AI-assisted coding features (autocomplete, inline suggestions, prompt based edits)
5. Noted what worked, what confused me, and what I'd do differently next time

---

## Where I Got Stuck
- Figuring out how extensions interact with each other inside Cursor
- Understanding *when* to prompt vs. just type (the boundary isn't obvious at first)

## How I Got Unstuck
- Read the actual docs (not just the README)
- Ran small experiments instead of trying to understand everything upfront

---

## Honest Takeaways
- AI tools are genuinely useful for speeding up repetitive or boilerplate work
- The quality of output depends heavily on how you frame the prompt
- First-time setup always takes longer than expected; that's normal

## Research Project (Step 2)

For the second step, I focused on analyzing LinkedIn organic content strategies for B2B SaaS.

### What I did:
- Selected 10 active creators who consistently produce high-performing content
- Analyzed their content style, structure, and recurring patterns
- Documented individual breakdowns in `/research/linkedin-posts`
- Created a synthesis of key insights in `/research/synthesis.md`

### Key Learning

From this research, I realized that strong content is not necessarily complex. The most effective creators focus on clarity, consistency, and making their ideas easy to consume.
